Here is a clear and simple comparison between Function Declaration (Definition) and Function Expression in JavaScript.

‚úÖ 1. Function Declaration (Function Definition)
‚úî Syntax:
function sayHello() {
  console.log("Hello!");
}

‚úî Key Features:
1. Hoisted ‚Üí You can call the function before it is defined.
2. Always creates a named function.
3. Load at the time of compilation.
4. Good for normal reusable functions.

‚úî Example:
greet(); // Works

function greet() {
  console.log("Good Morning!");
}

‚úÖ 2. Function Expression
‚úî Syntax:
const sayHello = function () {
  console.log("Hello!");
};


or arrow function:

const sayHello = () => console.log("Hello!");

‚úî Key Features:
1. NOT hoisted the same way ‚Üí You cannot call it before creation.
2. Can be anonymous (no name).
3. Function is created at runtime.

4. Commonly used in callbacks, closures, higher-order functions.

‚ùå Example (won't work):
greet();  // ‚ùå Error: greet is not a function

const greet = function () {
  console.log("Hello!");
};
