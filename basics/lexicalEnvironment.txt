A Lexical Environment is a structure that stores:
1. Local variables (let, const, function arguments)
2. Function declarations
3. A reference to the outer (parent) environment

It is created every time:
1. A function is called
2. A block executes (if, for, { })
3. A script or module starts

ğŸ“Œ Why is it called "Lexical"?
Because the environment is determined by where the code is written (lexically), not where it is executed. JavaScript looks at the physical placement of your code to decide the scope.
