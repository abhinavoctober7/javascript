Shadowing in JavaScript means:
A variable in an inner scope having the same name as a variable in an outer scope, so the inner one â€œshadowsâ€ (hides) the outer one.

JavaScript will use the nearest variable with that name.

The inner a = 20

ğŸ”¹ Shadowing with var
var is function-scoped, so shadowing behaves differently:
var x = 5;
if (true) {
  var x = 100;   // SAME variable (not shadowing!)
}
console.log(x); // 100
Here no shadowing happens because both x belong to the same function/global scope.

ğŸ”¹ Shadowing with let / const (Correct Shadowing)
let x = 5;
if (true) {
  let x = 20;    // shadowing
  console.log(x); // 20
}
console.log(x); // 5
